# AES-301: Engineering Session Closeout

## Status

Draft 0.1

## Purpose

Focused engineering work should end with a clear written checkpoint.

This standard defines a minimal closeout process for AES work sessions.

## Requirement

Before ending a focused session, record the current state of the work in the appropriate repository file.

## Checklist

- Record accepted decisions.
- List open questions.
- List deferred items.
- Commit new documents or code.
- Note known risks.
- Identify the next starting point.

## Rationale

A clear checkpoint preserves context and allows future work to resume from a known state.

## Guidance

Accepted decisions belong in ADRs, standards, design notes, or project checkpoints.

Ideas that are not yet accepted belong in a parking lot or research note.

Incomplete work should state what remains and where to resume.
