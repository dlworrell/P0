# Project Zero

## Status

Active

## Purpose

Project Zero establishes the engineering foundation for the Catalyst and Atarix ecosystem before substantial production code is written.

Its purpose is to preserve the early engineering philosophy, project structure, decision process, and documentation model in Git rather than leaving them only in conversation.

## Foundational Principle

First, Observe. Then, Understand. Then, Improve.

Observation precedes understanding. Understanding precedes improvement.

## Current Outcomes

- AES is the Atarix Engineering Standard.
- AES is the engineering governance repository.
- Just-a-Geek-LLC remains the organizational governance repository.
- repo_templates provides reusable project scaffolding.
- AEMS, Atarix, JAG, and future projects should inherit from AES.
- The Canon belongs inside AES unless a later ADR changes that decision.
- Documentation is part of engineering.
- Git is the source of truth; conversation is design discussion.
